#  项目名称
EdgeVib：面向工业预测性维护的边缘智能振动监测系统

---

# 1.项目背景

在工业设备（电机、泵、风机）中，振动是故障最关键的早期信号之一。
传统维护方式存在问题：

-  人工巡检成本高
-  无法提前预测故障
-  云端分析延迟高

因此，本项目构建一个：

 **“边缘AI + 工业采集 + 大数据分析”的预测性维护系统**

---

# 2.  项目目标

- 实现多节点振动数据采集（STM32）
- 实现边缘AI分析（ESP32 / 树莓派）
- 实现工业协议通信（RS485 / CAN / Ethernet）
- 实现大数据平台分析（Kafka + Spark）
- 实现Web监控系统（Java后端）

---

# 4.  系统总体架构

## 4.1 分层架构
[设备层]
STM32（DMF407）+ ADXL345 + 电机 + IO

[边缘层]
ESP32（AI分析） + 树莓派（网关 + Linux驱动）

[平台层]
Kafka + HDFS + Spark + Flink

[应用层]
Spring Boot + Vue


---

# 5.  硬件资源分配（重点）

## 5.1 STM32 DMF407（核心节点）

###  数据采集模块
- ADXL345（三轴振动）
- ADC（模拟量采集）
- 编码器接口（转速检测）

###  执行与控制模块（必须用上）
- 直流有刷电机（模拟设备运行状态）
- PWM调速（不同工况）
- 继电器 / MOS（控制设备开关）

###  工业IO（亮点）
- 隔离输入（工业传感器信号）
- 隔离输出（控制执行器）
- 光耦隔离（抗干扰）

###  通信接口
- RS485（Modbus RTU）
- CAN（工业总线）
- Ethernet（TCP/IP）

###  本地交互
- 蜂鸣器（报警）
- LED（状态）
- 按键（控制）

---

## 5.2 ESP32-S3（边缘AI节点）

- FFT振动分析
- TinyML模型（异常检测）
- WiFi上传
- UART与STM32通信

---

## 5.3 树莓派（网关 + Linux驱动）

###  Linux驱动（必须体现能力）
- I2C驱动（ADXL345读取）
- SPI驱动（扩展设备）
- GPIO驱动（控制IO）
- 字符设备驱动（自定义）

###  网关功能
- MQTT客户端
- 数据缓存（SQLite）
- 边缘AI增强分析（Python）

# 7.  大数据系统设计（重点加分）
由于学生环境限制，本项目采用 **本地虚拟机 + Docker 模拟企业级大数据平台**。

## 部署方式

- 使用 VMware / VirtualBox 部署 Ubuntu
- 使用 Docker Compose 启动大数据组件

## 7.1 数据流
STM32 → ESP32 → 树莓派 → Kafka → Spark → HDFS → Web
