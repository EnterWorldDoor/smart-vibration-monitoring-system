# 1. 项目背景

在工业生产环境中，旋转机械设备（如电机、风机、泵、压缩机等）在长期运行过程中容易出现以下故障：

- 轴承磨损
- 转子不平衡
- 联轴器不对中
- 机械松动
- 齿轮故障

传统维护方式通常采用：

1. 定期人工巡检
2. 定期停机维护

这些方式存在以下问题：

- 故障发现滞后
- 突发停机导致生产损失
- 维护成本高
- 缺乏实时监控

随着工业物联网（IIoT）和边缘计算技术的发展，通过 **实时振动监测 + 边缘AI分析** 可以实现 **预测性维护（Predictive Maintenance）**，提前识别设备异常，降低停机风险。

本项目旨在构建一个 **低成本的智能振动监测系统**，通过嵌入式节点采集设备振动数据，并在边缘网关上运行AI模型进行故障诊断。

---

# 2. 项目目标

本项目的主要目标是开发一个 **面向工业设备的预测性维护系统**，实现以下能力：

### 2.1 实时振动监测

系统能够持续采集设备振动信号，并实时传输到网关系统。

### 2.2 边缘智能诊断

在边缘网关上运行 AI 模型，对振动信号进行分析，实现设备故障预测。

### 2.3 故障预警

当系统检测到异常振动特征时，能够及时发出报警。

### 2.4 可视化监控

通过 Web Dashboard 展示设备运行状态、振动波形、历史数据和诊断结果。

### 2.5 可扩展的物联网架构

系统应支持多设备节点接入，具备工业 IoT 系统架构特征。

# 3. 项目要解决的问题

本系统主要解决以下工业问题：

## 3.1 设备故障难以及时发现

许多工业设备运行环境复杂，人工巡检频率低，容易错过早期故障信号。

解决方案：

- 实时振动采集
- 自动数据分析

---

## 3.2 设备停机造成生产损失

突发故障可能导致整条生产线停机。

解决方案：

- 预测性维护
- 提前预警

---

## 3.3 缺乏数据驱动的维护决策

传统维护依赖经验，而不是数据。

解决方案：

- 数据存储
- AI故障诊断

---

## 3.4 设备维护成本高

传统维护方式：

- 定期更换零件
- 不必要的停机维护

解决方案：

- 基于设备状态的维护（Condition-based Maintenance）
